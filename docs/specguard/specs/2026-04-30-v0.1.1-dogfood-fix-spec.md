# specguard v0.1.1 dogfood 修复 spec

**日期**：2026-04-30
**状态**：Draft，已用户拍板
**适用范围**：v0.1.0 dogfood 暴露的两个 blocker：plugin manifest 校验失败、`/specguard:init` 不能自动合并 hooks。

---

## ADR 级别决策识别（必填，不允许空）

### 改动点拆解

1. 删除 plugin.json 中 `commandNamespace: "sg"` 字段；接受 `/specguard:*` 命名空间。
2. commands frontmatter 给 `argument-hint` 加引号，使 YAML 合法。
3. `/specguard:init` 命令 prompt 中内嵌 hooks JSON snippet，并写出 `.specguard/hooks.snippet.json` 供用户手动合并；不再要求运行时定位 plugin 目录或自动改 `.claude/settings.json`。
4. init prompt 中按 layout 直接给出具体 paths，去掉对运行时模板渲染的隐含依赖。
5. plan dogfood 流程改为 `claude --plugin-dir dist/claude/<layout>` 单会话加载，README 同步。

### 五条硬条件匹配

| 改动点 | 接口语义 | 数据格式 | 跨模块依赖 | 外部依赖 | 推翻先前 |

| #2 frontmatter 引号 | - | - | - | - | - |

## 2. 验收标准

1. `claude plugin validate dist/claude/<layout>` 在三种 layout 下全部通过（warning 可接受）。
2. `uv run pytest` 全绿。
3. `claude --plugin-dir dist/claude/superpowers -p '/specguard:init --ai claude --spec superpowers' --permission-mode bypassPermissions` 在干净 worktree 上：
   - 创建 `docs/superpowers/design.md`、`decisions/{README,TEMPLATE}.md`、`specs/TEMPLATE.md`、`.specguard-version`
   - CLAUDE.md `<!-- specguard:start -->` 块注入
   - 写出 `.specguard/hooks.snippet.json`（内容与 plugin 渲染产物一致）
   - 不修改 `.claude/settings.json`（Claude Code 内置安全约束），但报告中给出手动 merge 指令
   - 不破坏既有 `docs/superpowers/specs/`、`plans/`、`CLAUDE.md` 顶部块以外内容
4. 用户按报告手动 merge `.claude/settings.json` 后，紧接其后 `/specguard:check` 报告 0 errors（可有 warnings）。
